# Mi Proyecto

Este es un proyecto de ejemplo que demuestra cómo usar Git y GitHub para gestionar un repositorio local y remoto. Incluye archivos Markdown con instrucciones detalladas sobre cómo trabajar con repositorios y cómo usar la terminal para navegar y crear archivos y directorios.

---

### Pasos para clonar y ejecutar el proyecto

1. Clona el repositorio en tu máquina local:
   ```bash
   git clone https://github.com/hacUPB/prog-2510-git-github-quintanamt.git

2. Navegar al directorio del proyecto:

   Una vez clonado, ingresa a la carpeta del proyecto:

   - `cd prog-2510-git-github-quintanamt`


3. Crea un directorio 
  - `mkdir mi_proyecto`

4. Crear las carpetas 
    
   - `mkdir src`
   -  `mkdir docs`
   - `mkdir images` 

5. En la carpeta 
 - `src ` Crear un archivo python agregar el código creado por el profesor con terminacion.py

    - `cd src `
     - `touch codigo.py `
   
    - `code .  `

### Nos abre el código en visual core studio copiamos y pegamos el código 

6. Empezamos 
 - `git add src `

7. Agregamosn el 
 - `git commit -m "Se agrego src donde esta el codigo python" `

8. - `git push -u origin main`

### .gitignore

9. Crear un archivo - ` .gitignore` y cúal es su función dentro de un proyecto de programación. Agrega el archivo - `.gitignore` a tu repositorio y escribe en él, el texto correspondiente para tu proyecto, según tu investigación. Pregunta al profesor en caso de dudas.

10. en el repositorio 
- `mi_proyecto` 

   - `touch .gitignore`

11. En la terminal revisar con: ` ls-la`si el archivo está ahí 

12. Cuando reviso que está lo abro con:
   - `code .gitignore`

Agrego la información que investigue y lo agrego al Git

13. Una vez creado 
   - `git add .gitignore`
  - ` git commit -m "Agregar archivo .gitignore"`
  - ` git push origin main `


### Punto Documentacion

14. Debo de ir a la carpeta - `docs`
   - `touch uso_consola.md `
  - ` touch repositorio_local.md`
   - `touch repositorio_remoto.md `

15. Con los documentos creados procedo a agregar toda la información en cada uno 
   - `code uso_consola.md`
   - `code repositorio_local.md `
  - ` code repositorio_remoto.md`

16. Para simplificar y avanzar agrego toda la carpeta docs que incluye los 3 archivos anteriores 

   - `git add .`
  - ` git commit -m "Primer iteam del proyecto"`
  - ` git push -u origin main- `

### Con toda la información abro la carpeta images 

17. Agrego una carpeta `images` que adjunta los pantallazos del trabajo realizado
